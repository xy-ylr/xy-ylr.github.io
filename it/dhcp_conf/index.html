DHCP是动态主机配置协议，一种集中对用户IP地址进行动态管理和配置的技术。适用于终端用户较多，甚至是用户随机入网、随机时长、但是用户数超过可分配IP个数的网络环境。

根据IP地址分配机制，可以由管理员手动分配；也可以由DHCP给用户动态分配一个有使用期限的IP地址；或者有DHCP给用户静态分配一个无限期使用的IP地址，这个类似于人工手动分配，但是可以避免一些人工配置的失误。

本文相关配置命令以H3C模拟器为准，华为设备配置可能略有不同。且各版本型号配置命令可能也有区别

参考资料：华三交换机DHCP配置、华为交换机DHCP配置

DHCP原理

这里只大概介绍最简单的一次性成功的工作原理，细节请另找资料阅读。

  1. 客户端发起广播(DHCP DISCOVER报文)，向网段内所有其他’用户‘申请IP：我需要一份工作(IP)，哪位老板给份工作(IP)…
  2. 网段内的DHCP服务器或者中继收到报文后会根据自身的DHCP地址池等情况给出回应(DHCP OFFER报文)：小伙子，我这边有份工作(IP)，钱多事少，快来吧…
  3. 客户端收到（可能有多个）服务器发来的报文，选择第一个DHCP服务器的报文，提取IP地址等，然后再次发起广播(DHCP REQUEST报文)：我收到了XX老板的职位邀请(IP)，准备上岗了，请老板确认一下…
  4. DHCP服务器收到DHCP REQUEST报文，确认无误后，再次给出回应(DHCP ACK报文)：好的，欢迎小伙子入职(IP)，好好表现…

最简模式-动态分配

如图所示，最简单的配置模式，就是局域网中所有终端用户都使用动态获取IP地址的方式，那就只需要在交换机上启用DHCP服务并配置IP地址池等功能就可以了。

<H3C>sys
[H3C]sysn server
# 启用dhcp服务
[server]dhcp enable
# 创建vlan 10
[server]vlan 10
[server-vlan10]qu
# 进入vlanif 10接口，配置ip地址并使接口工作在dhcp服务器模式
[server]int vl10
[server-Vlan-interface10]ip add 10.1.1.254 24
[server-Vlan-interface10]dhcp select server
[server-Vlan-interface10]qu
# 创建dhcp地址池vlan10，并配置ip地址池，网关和dns地址等
[server]dhcp ser ip-pool vlan10
[server-dhcp-pool-vlan10]network 10.1.1.0 mask 255.255.255.0
[server-dhcp-pool-vlan10]gateway-list 10.1.1.254
[server-dhcp-pool-vlan10]dns-list 60.191.134.196
[server-dhcp-pool-vlan10]qu
# 配置与终端互连的端口为access vlan10
[server]int ra g1/0/1 to g1/0/10
[server-if-range]port acc vl 10

如果局域网内有自建dns服务器，就用自建的dns服务器IP地址；如果没有，就用运营商的，或者其他公共dns地址。如果是普通外网的话不配置dns应该也可以，按照平时电脑上IP地址的经验来看，电脑会默认用网关做dns，网关实际没有配置dns相关功能，就会自动向上一级寻找，如果都没配置，最终也会找到运营商的dns服务器来解析域名什么的。

经过这些配置后，终端PC机等就能够自动获取到10.1.1.0网段的IP地址。如果还有其他接入交换机位于该核心交换机与终端之间，将接入交换机端口也配置access vlan10就好了。

实际上，如果局域网络很简单，此处交换机可以直接使用vlan 1来配置dhcp，那其他交换机就可以什么都不配置直接当傻瓜机来用。

静态绑定

可能网络中有搭建web服务器、ftp服务器、以及自建dns服务器等，这些服务端业务需要使用固定IP地址，我们也可以使用静态绑定的方式把相关业务IP地址固定下来。

<server>sys
System View: return to User View with Ctrl+Z.
[server]dhcp ser ip vlan10
# 采用静态绑定方式将IP地址与服务器网口mac地址绑定
[server-dhcp-pool-vlan10]static-bind ip 10.1.1.110 24 hardware-address 64e0-80c3
-0506
[server-dhcp-pool-vlan10]static-bind ip 10.1.1.111 24 hard 64e0-0415-0406

服务器mac地址可以从服务器网卡信息上看到，如果已经提前手动配置好了IP地址，那在交换机上运行命令dis arp也能看到。

备注：此处模拟器上web和ftp也是用的dhcp模式，没有手动配置ip，按照我对华三官方文档的理解，此处我设置了静态绑定之后，web或者ftp向dhcp服务器申请ip的时候，dhcp服务器应该会根据mac将我绑定的ip分配给他们，实际确实按照顺序给他们分别分配了10.1.1.4和10.1.1.5的ip。甚至我把web改为静态配置10.1.1.110后，使用dis dhcp server ip-in-use看到的还是属于Not used状态，只能两手一摊表示，**傻逼…** 后来又仔细看了下，发现通过dis mac-add看到的设备mac地址并不是出厂桥MAC地址，也就是说我静态绑定的时候mac搞错了-_-|||，绑定正确的mac确实是这种效果的。

IP不参与动态分配

如果没有配置静态绑定，那么虽然我们已经手动配置好了IP地址，但是当我们在交换机上运行命令dis dhcp server ip-in-use查看DHCP服务器已分配IP的时候，是看不到这些服务器IP地址的。这也就意味着，随着客户端的增加，这些IP（主机关机了的时候）有可能会被动态分配给别的终端使用，这时候我们也可以主动把一些IP地址配置为不参与动态分配。

2
3
4

<server>sys
System View: return to User View with Ctrl+Z.
[server]dhcp server forbidden-ip 10.1.1.110
[server]dhcp server forbidden-ip 10.1.1.111

调整IP地址租期

1
2
3
4
5

<server>sys
System View: return to User View with Ctrl+Z.
[server]dhcp ser ip-pool vlan10
# 设置IP地址租期为2分钟
[server-dhcp-pool-vlan10]expired day 0 hour 0 minute 2

可选配置，默认情况下IP地址租期为1天。如果是公共场合供他人无线连接上网，或者其他终端数很多且更换频繁的场景，可以把租期设置的短一些；如果企业内部办公电脑使用，基本不会有变化的，也可以考虑把租期延长。

DHCP Snooping

DHCP Snooping是DHCP的一种安全特性。

DHCP Snooping可以对交换机端口设置trust，以保证客户端从受信任的DHCP服务器获取IP。当客户端发出广播报文后，网络中的DHCP服务器会作出回应，从trust端口过来的DHCP报文会正常转发给客户端，而从其他未设置为trust的非信任端口(非法DHCP服务器)过来的DHCP报文会被丢弃。

一个最基本的好处就是，在终端接入交换机上设置了dhcp snooping之后，底下终端网络有人误接路由器，造成网络中多出来一个非法dhcp服务器，也只会造成局部（比如误接路由器的办公室办公室）无法上网，不至于影响到其他地方。

<H3C>sys
System View: return to User View with Ctrl+Z.
[H3C]sysn snooping
# 开启dhcp
[snooping]dhcp enable
# 开启dhcp snooping
[snooping]dhcp snooping enable
# 进入与dhcp服务器互连的口子，并配置dhcp snooping trust
[snooping]int g1/0/1
[snooping-GigabitEthernet1/0/1]dhcp snooping trust
[snooping-GigabitEthernet1/0/1]qu
# 创建vlan并将与终端互连的口子加入相应vlan
[snooping]vlan 10
[snooping-vlan10]qu
[snooping]int ra g1/0/1 to g1/0/10
[snooping-if-range]port acc vl 10

如果是有多个交换机串联的，那就将交换机上联方向去往dhcp服务的口子都设置为dhcp snooping trust。

DHCP中继

因为DHCP获取IP地址只能在同网段广播发送请求报文，所以如果DHCP服务器和客户端不在一个网段内，则客户端无法获取到IP地址。这个时候可以部署DHCP中继，中继会在接收到客户端发送的请求后，进行相应包装，然后转发给DHCP服务器，然后在收到DHCP服务器返回的报文后再转发给客户端。

### 配置dhcp和relay相关接口
## 配置dhcp相关接口
<dhcp>sys
System View: return to User View with Ctrl+Z.
# 创建vlan 10 20 30
[dhcp]vlan 10
[dhcp-vlan10]vlan 20
[dhcp-vlan20]vlan 30
# 使能vlanif30接口，并配置管理ip
[dhcp-vlan20]int vl30
[dhcp-Vlan-interface30]ip add 10.1.3.1 24
[dhcp-Vlan-interface30]qu
# 使能vlanif10接口，并配置IP作为vlan10网段网关
[dhcp]int vl10
[dhcp-vlan-interface10]ip add 10.1.1.1 24
[dhcp-vlan-interface10]qu
# 将连接终端相关端口配置为Access口，并放行vlan 10
[dhcp]int ra g1/0/2 to g1/0/10
[dhcp-if-range]port acc vl 10
[dhcp-if-range]qu
# 配置与relay互连的端口为Hybrid，放行vlan 30并untagged 
[dhcp]int g1/0/1
[dhcp-GigabitEthernet1/0/1]port link-type hybrid
[dhcp-GigabitEthernet1/0/1]port hybrid pvid vlan 30
[dhcp-GigabitEthernet1/0/1]port hybrid vlan 30 untagged
[dhcp-GigabitEthernet1/0/1]undo port hybrid vlan 1
## 配置relay相关接口
<relay>sys
System View: return to User View with Ctrl+Z.
# 创建vlan 10 20 30
[relay]vlan 10
[relay-vlan10]vlan 20
[relay-vlan20]vlan 30
# 配置与dhcp互连的G1/0/1口为Hybrid，放行vlan 30并untagged
[relay-vlan30]int g1/0/1
[relay-GigabitEthernet1/0/1]port link-type hybrid
[relay-GigabitEthernet1/0/1]port hybrid vlan 30 untagged
[relay-GigabitEthernet1/0/1]undo port hybrid vlan 1
[relay-GigabitEthernet1/0/1]port hybrid pvid vlan 30
[relay-GigabitEthernet1/0/1]qu
# 配置其他与终端互连的接口为Access，并放行vlan 20
[relay]int ra g1/0/2 to g1/0/10
[relay-if-range]port acc vl 20
[relay-if-range]qu
# 使能vlanif20接口，并配置IP作为vlan20网段网关
[relay]int vl20
[relay-Vlan-interface20]ip add 10.1.2.1 24
[relay-Vlan-interface20]qu
# 使能vlanif30接口，并配置管理IP
[relay]int vl30
[relay-Vlan-interface30]ip add 10.1.3.2 24

### 配置dhcp服务器
## 配置vlan10地址池
[dhcp]dhcp enable
[dhcp]int vl10
[dhcp-Vlan-interface10]dhcp select server
[dhcp-Vlan-interface10]qu
[dhcp]dhcp ser ip vlan10
[dhcp-dhcp-pool-vlan10]net 10.1.1.0 mask 255.255.255.0
[dhcp-dhcp-pool-vlan10]gat 10.1.1.1
[dhcp-dhcp-pool-vlan10]qu
# web服务器的ip已手动设定，不参与动态分配
[dhcp]dhcp server forbidden-ip 10.1.1.100
## 配置vlan20地址池
[dhcp]int vl20
[dhcp-Vlan-interface20]dhcp sel ser
[dhcp-Vlan-interface20]qu
[dhcp]dhcp ser ip vlan20
[dhcp-dhcp-pool-vlan20]net 10.1.2.0 mask 255.255.255.0
[dhcp-dhcp-pool-vlan20]gat 10.1.2.1
[dhcp-dhcp-pool-vlan20]qu

### 配置relay中继
[relay]dhcp enable
[relay]int vl20
# 该vlanif20接口的IP在前面已经配置号
[relay-Vlan-interface20]dhcp select relay
[relay-Vlan-interface20]dhcp relay server-address 10.1.3.1
[relay-Vlan-interface20]qu

### 配置两边路由
# 配置dhcp服务器上到vlan20客户端网段的路由
[dhcp]ip route-static 10.1.2.0 24 10.1.3.2
# 配置relay中继上的默认路由
[relay]ip route-static 0.0.0.0 0 10.1.3.1

因为dhcp服务器跟客户端不在同一个网段，所以需要路由让两边能够数据互通。

关于中继使用场景

关于DHCP中继的使用场景，其实没想明白有哪些情况下必须使用中继，或者是哪些情况下使用中继模式会更好。介绍里只是说当终端和dhcp服务器不在同一个网段的时候，可以使用DHCP中继手段来实现。但是既然终端的IP地址池在dhcp服务器上，那也就是说DHCP服务器上本来就有终端的网段信息的，限制也就只在dhcp服务器与中继互连的那部分端口了，只要把该互连端口配置成与终端同网段的，那dhcp服务器和终端不就在一个网段内了吗？

为什么非要卡住这个口子不让两边直接互通呢，是出于什么安全考虑吗？
