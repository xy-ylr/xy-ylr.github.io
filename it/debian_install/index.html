安装Debian系统后的基础配置

debian系统安装完成后，总还是需要进行一些个性化配置，或者安装一些个人喜欢的常用软件之后，才能爽起来，本文记录一些相关操作，免得以后重装系统的时候重复找资料。

本文系统环境是debian 11的镜像安装，然后升级为debian 12，带图形界面。

安装过程省略，图形化安装已经非常简单了，看着来就行了。

网络设置

图像界面的网络设置跟windows也没多少区别，网线一接，要么自动获取，要么手动设置ip。

如果是笔记本，一开始可能无线网卡驱动没有安装，无法使用wifi。

如果是minimal安装，没有图形界面，那需要通过命令行进行配置。

2

# 通过终端命令行配置网络

sudo

在安装Debian的时候，除了root用户，还有创建一个普通用户，手动为该用户赋予sudo权限。先用root用户登录，然后执行如下命令

1

# usermod user -a -G sudo

将用户user加入sudo用户组。

别的发行版linux可能不是sudo用户组，更多细节请参考man sudo，/etc/sudoers

命令补全

刚安装完成系统的时候，并没有安装bash shell的命令补全，这样子在终端输入命令的时候，就无法通过Tab键自动补全命令，使用起来就会很不友好，所以我们需要额外安装一个软件包bash-completion

1

$ sudo apt install -y bash-completion

更改国内源

如果使用debian官方源的话，后续安装、更新软件什么的，可能速度会比较感人，所以需要改成国内的。

其实在图形化安装系统的时候，就有提示选择国内镜像源，可以根据提示从清华、中科大、163等镜像源中间选一个，不过这个好像需要联网才能配置成功。

如果在安装的时候没有改成国内的源，那这里就需要手动改一下了，这里给几个参考，本人用的中科大的镜像源。

清华大学开源软件镜像站-Debian软件仓库镜像使用帮助，阿里巴巴开源镜像站-Debian镜像使用帮助，中科大开源镜像站-Debian源使用帮助

核心都是将 /etc/apt/sources.list 文件中 Debian 默认的源地址 http://deb.debian.org/ 替换为镜像地址

注意，上述使用帮助，清华、阿里提供的参考配置，都是追求某一特定版本的源，而中科大给出的参考配置，追求的是最新稳定版。

比如当前2023年，Debian官方最新稳定版（Stable）是Debian 12，代号Bookworm；上一个稳定版（Old Stable）是Debian 11，代号bullseye。

使用中科大的参考配置，当后续Debian 13发布的时候，你可以运行sudo apt full-upgrade命令，把你的系统直接升级为Debian 13。而清华、阿里的无法直接通过这条命令直接升级到Debian 13。

造成这种区别的原因在于中科大的参考配置里用的stable，而不是版本代号bookworm，所以你也可以使用别的镜像源，然后配置里也使用stable而不用代号。

deb http://mirrors.ustc.edu.cn/debian stable main contrib non-free non-free-firmware
deb-src http://mirrors.ustc.edu.cn/debian stable main contrib non-free non-free-firmware
deb http://mirrors.ustc.edu.cn/debian stable-updates main contrib non-free non-free-firmware
deb-src http://mirrors.ustc.edu.cn/debian stable-updates main contrib non-free non-free-firmware

deb http://mirrors.ustc.edu.cn/debian-security/ stable-security main non-free contrib
deb-src http://mirrors.ustc.edu.cn/debian-security/ stable-security main non-free contrib

另外，为了更及时地获得安全更新，也可以考虑最底下的安全更新软件源继续使用Debian官方源，因为镜像站往往有同步延迟。参考https://www.debian.org/security/faq.en.html#mirror

The purpose of security.debian.org is to make security updates available as quickly and easily as possible.

Encouraging the use of unofficial mirrors would add extra complexity that is usually not needed and that can cause frustration if these mirrors are not kept up to date.

更新系统

当国内源配置完成后，就顺便更新一下系统，因为iso系统镜像里的软件可能已经不是最新稳定版了。

1
2
3

$ sudo apt update
$ sudo apt upgrade
$ sudo apt autoremove

第一条命令是更新软件包索引；第二条命令是安装/升级软件；第三条命令是删除一些不常用软件。

其他的还有apt search、apt install等，请通过man apt或者apt -h获取帮助。

检查软件源

查看/etc/apt/sources.list文件中是否有类似如下内容：

1
2

# Debian Bookworm
deb http://deb.debian.org/debian/ bookworm main contrib non-free non-free-firmware

主要是看有没有“contrib”、“non-free”、“non-free-firmware”三个组件，实际上之前更改国内源的时候已经配置好了。

声卡驱动

1
2

xy@xy:~$ lspci | grep Sound
0000:00:1f.3 Multimedia audio controller: Intel Corporation Tiger Lake-LP Smart Sound Technology Audio Controller (rev 20)

当初具体解决过程有点记不清了，debian-声卡文档没咋看明白。一开始发现电脑虽然没声音，但是声音设置什么看起来都是正常的，给人的感觉就是声卡驱动已经安装成功了的。网上搜索“debian没有声音”，看了不少人的博客文章，全都是无效方案。

直到看到debian中文论坛-[新手求助]请问各位大佬debain11没有声音怎么解决，提示no soundcards found…，以及其引用的文章InstallingDebianOn/Acer/Aspire 5 A515-56 - Debian Wiki，发现是缺少了firmware-sof-signed，安装之后就有声音了。

但实际上还是没搞明白文章里的这个结果是怎么得出来的，只能说根据lspci | grep Audio显示的硬件，在debian软件包搜索中根据描述信息搜索“intel audio"，firmware-sof-signed软件包确实是排在第一位。

无线网卡驱动

当初具体解决过程有点记不清了，网上看的别人的文章。

1
2

xy@xy:~$ lspci | grep Network
0000:00:14.3 Network controller: Intel Corporation Wi-Fi 6 AX201 (rev 20)

通过以上输出结果可以知道我们的无线网卡是Intel AX201。debian无线网卡官方文档中没有找到对应的型号，只能初步判断应该是需要iwlwifi，不过intel官网英特尔®无线适配器的 Linux* 支持可以确认ax201无线网卡确实是需要iwlwifi软件包。通过如下命令安装驱动：

1

xy@xy:~$ sudo apt update && apt install firmware-iwlwifi

配置网络时间同步

Debian默认没有配置ntp时间同步，这样当笔记本装有双系统的时候，切换到win系统使用一段时间之后，再切回来，好像时间就会不准。

根据官方文档，可以安装ntp软件包，Debian 12以上的版本，更推荐使用ntpsec包

1

$ sudo apt install -y ntpsec

安装完成之后，系统就会自动同步网络时间。

ntpsec的配置文件是/etc/ntpsec/ntp.conf，可以发现时间是从debian.pool.ntp.org同步过来的，如果不喜欢也可以改成中国的时间服务器，或者其他的时间服务器，见ntp时间服务器池

1
2
3
4
5
6
7

# pool.ntp.org maps to about 1000 low-stratum NTP servers.  Your server will
# pick a different set every time it starts up.  Please consider joining the
# pool: <https://www.pool.ntp.org/join.html>
pool 0.debian.pool.ntp.org iburst
pool 1.debian.pool.ntp.org iburst
pool 2.debian.pool.ntp.org iburst
pool 3.debian.pool.ntp.org iburst

顺便，阅读Debian官方wiki-日期时间可以发现，双系统时间不准的原因是：Debian默认保持硬件时钟与UTC时间一致，而Windows系统是保持硬件时间与当地时区的时间一致。所以从Windows系统切回来的时候，Windows将当地时区（东8区）的时间写入硬件时钟，而Debian把这个时间当作UTC时间来读取，然后显示的时候再一次增加时区偏移量，结果就是Debian的时间比实际要快8个小时。

解决方案就是修改配置文件/etc/adjtime，告诉Debian硬件时钟保存的是当地时间。这个配置文件是由hwclock产生的，一开始可能并不存在这个文件，可以先运行一次sudo hwclock --systohc，然后再将最后一行的UTC改为LOCAL。

1
2
3

0.000000 14602224559 0.000000
1460224559
LOCAL

关于该文件的含义，以及hwclock命令的更多用法，请参考man hwclock或者阅读在线版帮助文档
