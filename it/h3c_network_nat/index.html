简单理解，NAT是网络规模越来越大，而IPv4地址资源却越来越少，为了使两个不同的网络（公网和私网）连通，而将网络报文中的源或者目的IP地址进行转换的一种技术。所以简单的分类，NAT分为源NAT、目的NAT、源目的双向NAT。

由于NAT技术将数据报文中的IP地址进行了转换，一定程度上实现了对外隐藏内部网络拓扑、内部服务器真实IP地址等功能，可以杜绝部分网络攻击，提高网络安全性；也由于对外隐藏了内部网络拓扑，所以当内部网络结构调整的时候，外部网络不需要跟着变动，某种程度上也算是方便了网络管理。

基本介绍

源NAT

源NAT就是内网需要访问外网，在发起访问请求的时候，将内部网络的源IP转换成外网IP。

目的NAT

当内网有部分服务需要对外提供访问，因为服务部署在内网，外部网络本身是无法直接连通的，所以我们可以把外网IP映射到相应的内部服务器IP，这样当外网用户访问我们的外网IP的时候，请求就会自动转换并发送到配置好的对应业务服务器。

双向NAT

双向NAT指的是在转换过程中同时转换报文的源信息和目的信息。双向NAT不是一个单独的功能，而是源NAT和目的NAT的组合。双向NAT是针对同一条流，在其经过设备时同时转换报文的源地址和目的地址。双向NAT主要应用在外网用户访问内部服务器和私网用户访问内部服务器两个场景。

以上是华为的分类方法，华三分法是静态转换、动态转换、内部服务器等，叫法虽然不一样，但本质上nat原理应该是一样的。

## 创建策略，默认拒绝所有
[fw]security-policy ip
# 创建策略名称为deny_any
[fw-security-policy-ip]rule 0 name deny_any
# 策略动作为拒绝
[fw-security-policy-ip-0-deny_any]action drop
# 策略涉及的服务为any
[fw-security-policy-ip-0-deny_any]service any
## 创建策略，用于放通ping功能
[fw-security-policy-ip-0-deny_any]rule name ping_test
[fw-security-policy-ip-1-ping_test]service ping
[fw-security-policy-ip-1-ping_test]action pass
# 移动策略
[fw-security-policy-ip]move rule 1 before 0

rule的时候可以指定数字，比如rule 0 name ...，也可以省略数字，让系统自动生成。

rule 0的deny_any策略，其实就是防火墙的默认功能，拒绝所有流量经过防火墙，此处再显性配置一条。

rule 1的ping_test策略，用于放通任意ping功能，但是由于这条策略是在deny_any后面，所有流量匹配到默认拒绝所有之后，这条策略就匹配不到，无法生效了，所以最后需要move rule 1 before 0将之往前移动一下，后续再新增什么策略，也一样需要至少移动到deny_any策略之前。

到了这一步，我们的物理主机可以ping通防火墙，nw、ww侧的主机和交换机也都可以ping通防火墙了。

## 创建策略放行mgmt
[fw]sec ip
[fw-security-policy-ip]rule name mgmt_pass_http
[fw-security-policy-ip-2-mgmt_pass_http]description pass mgmt to FW for http,tel
net
# 设定源安全域
[fw-security-policy-ip-2-mgmt_pass_http]source-zone mgmt
# 设定目的安全域
[fw-security-policy-ip-2-mgmt_pass_http]destination-zone Local
# 设定服务
[fw-security-policy-ip-2-mgmt_pass_http]service http
[fw-security-policy-ip-2-mgmt_pass_http]service https
[fw-security-policy-ip-2-mgmt_pass_http]service telnet
# 设定策略动作为允许
[fw-security-policy-ip-2-mgmt_pass_http]action pass
# 移动策略位置
[fw-security-policy-ip]move rule 2 be 0

最后再配置放行nw、ww安全域之间的telnet功能

[fw]sec ip
# 新建策略nw_ww_telnet，放行nw域到ww域的telnet
[fw-security-policy-ip]rule name nw_ww_telnet
[fw-security-policy-ip-3-nw_ww_telnet]source-zone nw
[fw-security-policy-ip-3-nw_ww_telnet]destination-z ww
[fw-security-policy-ip-3-nw_ww_telnet]service telnet 
[fw-security-policy-ip-3-nw_ww_telnet]action pass
[fw-security-policy-ip-3-nw_ww_telnet]qu
[fw-security-policy-ip]move rule 3 be 0
# 新建策略ww_nw_telnet，放行ww到nw的telnet
[fw-security-policy-ip]rule name ww_nw_telnet
[fw-security-policy-ip-4-ww_nw_telnet]source-z ww
[fw-security-policy-ip-4-ww_nw_telnet]destination-z nw
[fw-security-policy-ip-4-ww_nw_telnet]service telnet 
[fw-security-policy-ip-4-ww_nw_telnet]act pass
[fw-security-policy-ip-4-ww_nw_telnet]qu
[fw-security-policy-ip]move rule 4 be 0

配置到这里，nw、ww安全域已经可以相互telnet了，简单验证一下，如下图所示，ww交换机已经成功登录到了nwhx交换机，前期准备工作就算是完成了。

目的NAT

如果内网有ftp、web等服务器需要开放给外网访问，可以配置目的Nat，将内网IP映射到外网IP，让外网用户访问相应的外网IP就能转换到内网对应的服务器上。内网服务器nat就是这么一种目的nat。

基本配置

2
3

<fw>sys
[fw]int g1/0/1
[fw-GigabitEthernet1/0/1]nat server global 10.10.10.254 inside 1.1.1.11 counting

添加端口限制

而在实际使用过程中，我们可能会根据内部服务器提供的服务，来进一步限定nat的端口号、协议类型。

比如刚才案例中，我们限定为telnet服务，23端口，那么刚才的命令就会变成如下

常用显示和维护命令

查看所有nat相关：dis nat all

查看当前nat会话信息：dis nat se ver

查看静态nat配置：dis nat static

查看内部服务器：dis nat server

查看出方向动态nat：dis nat outbound

查看入方向动态nat：dis nat inbound

其他还有一些命令请通过dis nat ?补全命令自行查看。

总结

大部分情况下的nat需求，仅单方面配置源nat或者目的nat就够了，比如：

  1. 如果内网有少量用户需要nat访问外网，可以配置出方向静态nat（源nat）。
  2. 如果内网有大量用户需要nat访问外网，那么可以配置出方向动态nat（源nat）。
  3. 而如果内网有web、ftp等业务需要对外开放，那么可以配置内部服务器nat（目的nat）。

再复杂一些的环境，就需要根据实际需求，同时配置出方向静态nat、入方向静态nat、内部服务器nat、出方向动态nat、入方向动态nat中的几种来实现双向nat了，比如：

  1. 内网有web、ftp等服务器需对外开放，同时其他内网用户要求也可以通过nat地址访问这些内网服务器。那么可以同时配置内网服务器nat、出方向动态nat，并启用nat hairpin功能。
  2. 如果外部服务器绑定有域名，且外部服务器的IP与内网用户IP重叠，内网用户要nat之后通过域名访问外部服务器。那么需要配置出方向动态nat、入方向动态nat，同时还要开启dns的nat alg功能。
  3. 如果内外网有业务往来，两边都有可能向对方发起业务访问请求。
    1. 正常情况下，如果两边都有边界防火墙等可以配置nat，且两边的IP地址不重叠，那边两边分别配置目的nat（内部服务器）就可以了。
    2. 如果只有一边可以配置nat，那么可以配置内部服务器nat、入方向动态nat或者入方向静态nat。
    3. 如果只有一遍可以配置nat，且两边IP地址重叠，那么还需要再配置出方向动态nat或者出方向静态nat。
  4. 可能还有其他一些特殊情况，具体情况具体分析吧……
