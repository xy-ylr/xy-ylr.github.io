系统辅助工具

命令行补全

有些linux发行版安装完成后, 在命令行输入部分命令再按Tab键无法实现命令自动补全, 这是由于缺少相应的扩展包, 使用以下命令安装, 然后重新打开命令行窗口就可以了.

centos:

1

yum install-y bash-completion

ubuntu:

1

apt-get install -y bash-completion

yum provides [service]

hostname

hostname [new_name]

vi /etc/hostname

lsb_release -a 查看linux发行版信息

cat /proc/version 查看内核版本信息, 其他还有cpuinfo, meminfo, swaps等

uname -a 显示系统信息

uname -r 显示内核版本信息

uname -n 显示主机名称

  1. df -h 查看磁盘空间使用情况, 单位大小显示为G/M/K等自适应

    -T 显示文件系统信息

  2. who 显示当前登录的用户信息.

  3. last 显示最近登录记录, 仅供参考, 黑客可能篡改过

  4. history 显示最近执行过的10条命令记录, 来源~/.bash_history

系统时间

date "+%Y-%m-%d %H:%M:%S" 按照指定格式显示当前系统时间, 不加参数以默认格式显示.

查看时间、时区等信息

timedatectl

设置时区

cp /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

timedatectl set-timezone Asia/Shanghai

修改系统时间

date -s "20200220 20:20:20"

timedatectl set-time 2024-05-18 设定日期，也可以设定时间

timedatectl set-ntp true 启用ntp同步（ntp服务器地址需额外指定，安装ntp服务）

# grep反向匹配，使用-v参数
# 排除指定字符串
grep -v 'strings' file.txt
# 排除指定单词 -w表示匹配单词
grep -vw 'word' file.txt
# 排除多个 -e 
grep -v -e 'string1' -e 'string2' file.txt
# 或者新建文件，每行一个排除项
grep -v -f exclude.txt file.txt
# 包含指定内容，同时不包含指定内容
grep 'string3' file.txt | grep -v -e 'string1' -e 'string2'
grep -v -e -e 'string1' -e 'string2' <(grep 'string3' file.txt)
# 在指定目录搜索内容的时候排除特定文件
grep -r --exclude='*.log' 'search-string' /path/
# 搜索的时候排除指定目录
grep -r --exclude-dir='exclude-dir' 'search-string' /path/

压缩解压缩

zip

unzip [-d path] test.txt.zip 解压缩zip文件[到指定目录]。

其他参数，-l查看压缩包中的文件列表；-v查看压缩包文件列表详细信息；

tar.gz

tar -xzvf filename.tar.gz

-x解压tar包；-z使用gzip来解压；-v显示解压过程，可省略；-f文件名。

文件下载

wget -c URL 从URL地址下载文件，支持断点续传，添加-O参数可以设置本地文件名；添加-x参数可以在本地强制创建目录保存到指定位置。

curl -o [file_name] URL 从URL地址下载文件并重命名。大写的-O参数表示使用原文件名下载保存。

补充：wget和curl如果以原文件名下载保存，好象是以URL连接中的最后一节作为文件名来保存，跟通过web页面下载时的文件名可能有区别。

========================================

  1. du -h -d 1 显示当前目录各文件及子目录的磁盘使用量, -d N表示递归显示的目录层数为N, 默认列出所有.

    -c 显示当前目录占用磁盘空间的归总数据.

  2. stat test.txt 查看一个文件的详细信息, 包括Inode, Device, Access, Modify, Change等.

  3. file test.txt 查看一个文件的文件类型.

  4. touch -t 202108091022 test.txt 修改一个文件的创建时间, stat命令里的Access.

  5. cp file1 newfile 复制file1到当前目录下newfile.

    mv file1 dir/newfile 移动当前目录下file1文件到dir子目录并重命名为newfile.

    rm -rf /test/ 删除目录下所有文件及子目录, -r表示循环递归, -f表示强制删除.

  6. wc test.txt 统计一个文件的行数, 单词数, 字节数.

  7. more +20 /var/log/messages 从第20行开始查看文件, 可以向下滚动查看内容并往回看, 但是前20行内容看不到.

  8. less +20 /var/log/messages 跟more命令类似, 但是可以往上查看前20行的内容.

  9. cat /dev/null > test.txt 清空文件内容, 或者新建空文件, >表示将文件内容替换为指定内容.

  10. echo "test" >> test.txt 向文件末尾追加内容"test”, >>表示追加内容.

  11. head -5 test.txt 显示文件头部的前5行内容, 如果不加参数, 默认显示10行.

  12. tail -n 10 /var/log/messages 显示文件尾部10行内容.

    -c 显示文件尾部指定字节数的内容.

    -f 实时刷新内容, 使用Ctrl+c退出.

用户管理篇

sudo usermod -aG <groupname> $USER 将当前用户追加到某个用户组。-aG两个参数配合使用，如果只使用-G参数，是将用户的辅助组替换为指定用户组。

日志篇

  1. journalctl -x 查看日志, 输出的是早期的日志

    -xe 跳到底部查看日志, 能看到最新的日志记录.

    -u 查看指定用户(服务)的日志. 比如journalctl -u cron.service
