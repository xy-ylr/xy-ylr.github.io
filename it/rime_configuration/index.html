Rime安装与基本配置

【注意】本文最后更新于 2022-11-23 ，文中内容可能已过时，请谨慎使用。

Rime输入法的安装与一些基本配置, 比如添加第三方词库, 快捷输入日期时间星期, 调整候选词为横排以及个数等. 本文主要针对Windows系统(小狼毫)和Ubuntu系统IBus框架(中州韵).

安装Rime

windows系统直接Rime官方网站下载安装.

使用IBus输入法框架的linux发行版, 参考官方文档-RimeWithIBus.

使用Fcitx输入法框架的linux发行版, 可以使用fcitx-rime, 该软件由Fcitx官方团队开发和维护. 参考文章 Linux 下的输入法 fcitx vs ibus

选择输入法方案

安装完成之后就可以开始使用了, Rime自带’明月拼音(繁体)’, ‘明月拼音简体’, ‘五笔’等输入方案.

我们可以通过快捷键Ctrl+`唤出方案选单, 选择朙月拼音-简化字输入方案, 这个类似于"搜狗拼音输入法”. 不过Rime本身自带词库比较小, 使用起来可能不如搜狗拼音输入法友好. 但好在它自定义功能很强大, 我们只需要再微调一下, 就会好很多.

前置知识点

必知必会

此处简单介绍部分配置文件及所在位置, 更多内容请看官方文档-必知必会-Rime中的数据文件分布及作用

先上总结: 自定义配置简单理解就是根据原有配置文件(后缀名为.yaml或者.schema.yaml), 在用户文件夹下创建同名配置文件(后缀名为.custom.yaml), 比如原有的default.yaml, 新建default.custom.yaml; 原有luna_pinyin_simp.schema.yaml, 新建luna_pinyin_simp.custom.yaml. 然后在新文件中写入设定, 以给源配置文件打补丁的方式覆盖默认配置.

共享文件夹: 安装Rime时自带的预设输入方案的源文件, 不建议改动, 会导致输入法升级版本的时候失败, 或者改动丢失.

小狼毫(Windows系统): "安裝目錄\data"

中州韵(ibus-rime): /usr/share/rime-data/

用户文件夹: 输入法运行时保存的数据, 安装信息, 用户词典, 用户自定义的配置文件等, 都放在此处. 官方文档-用户文件夹

小狼毫(Windows系统): %APPDATA%\Rime, 是一个隐藏文件夹, 实际位置可能是C:\Users\admin\AppData\Roaming\Rime

中州韵(ibus-rime): ~/.config/ibus/rime/, (0.9.1 以下版本爲 ~/.ibus/rime/；fcitx5 为 ~/.local/share/fcitx5/rime/).

** 文中后续提到的用户文件夹和共享文件夹具体路径参考此处介绍

用户文件夹中部分文件说明, 注意有些配置文件默认并不存在, 需要我们手动创建.

installation.yaml: 安装信息文件.

user.yaml: 用户状态信息文件.

default.yaml: 全局设定文件. 如果用户想要对全局设定进行修改, 在default.custom.yaml文件中进行.

<方案标识>.schema.yaml: 预设输入方案副本, 比如luna_pinyin_simp.schema.yaml是朙月拼音简体字输入法, 如果要对该输入法的设定进行调整, 则在新建文件luna_pinyin_simp.custom.yaml中进行.

*.dict.yaml: 用户自制词典文件.

部署调试

当我们通过配置文件修改配置后, 需要重新部署才能生效.

小狼毫可以通过鼠标右键单击右下角输入法图标, 然后左键单击选择重新部署; 或者开始菜单->小狼毫输入法->重新部署.

中州韵可以通过点击输入法状态栏上的部署按钮完成; 或者命令行方式触发:

1

touch ~/.config/ibus/rime/; ibus restart

如果部署之后没有实现预想的效果, 可以查看日志是否有报错等信息. 参考官方文档-关于调试:

不同操作系统的日志文件位置:

【中州韵】/tmp/rime.ibus.*
【小狼毫】%TEMP%\rime.weasel.*
【鼠须管】$TMPDIR/rime.squirrel.*
各发行版的早期版本用户文件夹/rime.log

按照日志的级别分为INFO /信息、WARNING /警告、ERROR /错误。后两类应重点关注，如果部署后不可用或输出与设计不一致，原因可能在此。

开始配置

候选词个数和方向

Rime默认给出的候选词是纵向排列5个, 习惯了横向排列, 且5个候选词太少.

用户文件夹下新建default.custom.yaml

1
2
3

patch:
  "schema_list/schema": luna_pinyin_simp
  "menu/page_size": 8         # 候选词个数

新建rime.custom.yaml(小狼毫是weasel.custom.yaml)

1
2

patch:
  "style/horizontal": true    # 候选词横向

注意这个patch, 也就是打补丁的意思, 前面不要有空格, 且每个文件中只需顶部出现一次就够了, 如果后面还有其他配置也是在相同的文件中设定, 无需重复输入patch.

中英文切换快捷键

Rime默认设置的shift键切换中英文效果与常用的搜狗输入法效果不一样, 如果习惯了搜狗的切换方式, 可以对其进行修改, 以下是根据个人使用习惯做的调整:

1
2
3
4
5
6
7

patch:
  ## 中英文切换快捷键设置，默认设置见default.yaml
  # noop：屏蔽按键； inline_ascii：临时英文，按enter上屏后重新变回中文；
  # commit_text：中文上屏，并切为英文； commit_code：英文字符上屏，并切为英文。
  "ascii_composer/switch_key/Shift_L": inline_ascii
  "ascii_composer/switch_key/Shift_R": commit_code
  "ascii_composer/switch_key/Control_L": commit_text

上述修改的配置文件是luna_pinyin_simp.custom.yaml, 针对的方案是明月拼音简化字, 如果你使用的是别的方案, 请尝试新建对应的<方案名>.custom.yaml文件.

网上的教程都是说在default.custom.yaml中修改, 但是经本人测试, 无论是windows系统下的小狼毫, 还是ubuntu下的中州韵ibus-rime, 在default.custom.yaml中配置中英文切换的快捷键都无法生效. 甚至直接在default.yaml中修改也无法生效, 原因未知.

快捷输入当前日期时间

function date_translator(input, seg)
    if (input == "dd") then
       --- Candidate(type, start, end, text, comment)
       yield(Candidate("date", seg.start, seg._end, os.date("%Y-%m-%d"), ""))
       yield(Candidate("date", seg.start, seg._end, os.date("%Y年%m月%d日"), ""))
    end
   if (input == "ww") then
      arr = {"一","二","三","四","五","六"}
      arr[0] = "日"
      yield(Candidate("date", seg.start, seg._end, os.date("%Y-%m-%d 星期"..arr[tonumber(os.date("%w"))]), ""))
   end
   if (input == "tt") then
      yield(Candidate("date", seg.start, seg._end, os.date("%Y-%m-%d %H:%M:%S"), ""))
      yield(Candidate("date", seg.start, seg._end, os.date("%H:%M:%S"), ""))
   end
end
 
--- 过滤器：单字在先
function single_char_first_filter(input)
   local l = {}
   for cand in input:iter() do
      if (utf8.len(cand.text) == 1) then
         yield(cand)
      else
         table.insert(l, cand)
      end
   end
   for i, cand in ipairs(l) do
      yield(cand)
   end
end

然后<方案名>.custom.yaml中添加配置:

1
2
3
4

patch:
  # 快捷输入当天日期时间等
  "engine/translators/+":
    - lua_translator@date_translator

重新部署后, 在<方案名>.schema.yaml源文件中会多出来一行代码 - "lua_translator@date_translator", 那就是成功了, 可以使用tt, dd, ww进行测试以下.

扩展外部词库

前面说过, Rime自带的词库规模比较小, 很多词组打不出来, 不停翻找候选词的话会非常影响输入效率. 为了提高打字体验, 我们需要引入一些外部词库, 比如搜狗标准细胞词库等.

搜狗输入法有官方细胞词库, 也有根据用户输入习惯形成的用户词库. Rime也一样, 有固态词典和用户词典, 相关介绍参考百度贴吧-rime吧:〔新手推荐敎程〕关于导入词库及「深蓝词库转换」的正确操作方法前面十几楼大佬的科普.

此处以明月拼音简化字输入方案添加搜狗标准词库为例.

luna_pinyin_simp.custom.yaml文件添加如下内容:

1
2
3

patch:
  # 新增外部词典
  "translator/dictionary": luna_pinyin.extended

用户文件夹下新建文件luna_pinyin.extended.dict.yaml, 该文件作为外部词典总控制文件, 参考内容如下:

# Rime dictionary
# encoding: utf-8
# 明月拼音扩展词典总控制文件，用于引入其他扩充词库, 自定义词汇等

---
name: luna_pinyin.extended
version: "2022.11.16"
sort: by_weight
use_preset_vocabulary: true
# 补充词库，不需要的直接注释即可。包括明月官方词库、搜狗标准词库、股票名词、快递/地址、中国旅游风景名胜、
import_tables:
  - luna_pinyin
  - sogou.standard
  # - sogou.gupiao
  # - sogou.kuaidi
  # - sogou.lvyou
...

## 以下补充自定义词汇，
## 词汇[Tab]拼音[Tab]词频
特仑苏	te lun su	100
苏轮特	su lun te	100

注意name部分,以及该文件的文件名, 需要与luna_pinyin_simp.custom.yaml中"translator/dictionary": luna_pinyin.extended定义的保持一致.

注意import_tables部分, luna_pinyin为Rime输入法明月拼音官方词库, sogou.standard就是我们即将添加的搜狗标准词库.

可以在底部添加一些偏门的自定义词汇, 重新部署后用于测试扩展词库是否添加成功. 自定义词汇分为三部分, 词汇, 拼音, 词频, 三者之间用Tab键隔开. 最主要的是词汇部分, 另外两个可以省略.

官方下载搜狗标准词库, 官方下载深蓝词库转换工具转换为rime拼音linux版, 重命名转换得到的Rime扩展词库文件为sogou.standard.dicy.yaml, 然后复制到用户文件夹下, 并在文件开头添加如下内容:

重新部署输入法之后, 添加搜狗标准词库应该就成功了, 同样的方法就可以添加其他需要的扩展词库. 词库文件可以去搜狗输入法等网站下载并转换获得, 或者直接使用其他大佬制作的现成的扩展词库, 比如xiaoTaoist
