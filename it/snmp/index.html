SNMP是简单网络管理协议，用以监测连接到网络上的设备是否有任何引起管理上关注的情况。SNMP采用轮询机制，提供最基本的功能集，适合小型、快速、低价格的环境使用，以用户数据报协议（UDP）报文为承载，因而受到绝大多数设备的支持，同时保证管理信息在任意两点传送，便于管理员在网络上的任何节点检索信息，进行故障排查。

随着时代发展，网络越来越庞大，网络设备数量和种类也越来越多、所处位置到处都是，设备厂商也越来越多，提供的管理接口可能也各不相同，导致网络管理员对相关设备的管理维护也随之越来越困难。

在这种情况下大佬们就整出了SNMP，利用网络管理网络：

  • 网络管理员可以利用SNMP平台在网络上的任意节点完成信息查询、信息修改和故障排查等工作，工作效率得以提高。
  • 屏蔽了设备间的物理差异，SNMP仅提供最基本的功能集，使得管理任务与被管理设备的物理特性、网络类型相互独立，因而可以实现对不同设备的统一管理，管理成本低。
  • 设计简单、运行代价低，SNMP采用“尽可能简单”的设计思想，其在设备上添加的软件/硬件、报文的种类和报文的格式都力求简单，因而运行SNMP给设备造成的影响和代价都被最小化。

参考：华为IP知识百科-什么是SNMP、华为S12700交换机SNMP配置。

SNMP基础

SNMP版本

目前SNMP有三种版本：SNMPv1，SNMPv2c和SNMPv3。

  • SNMPv1：SNMP的第一个版本，它提供了一种监控和管理计算机网络的系统方法，它基于团体名认证，安全性较差，且返回报文的错误码也较少。它在RFC 1155和RFC 1157中定义。
  • SNMPv2c：第二个版本SNMPv2c引入了GetBulk和Inform操作，支持更多的标准错误码信息，支持更多的数据类型。它在RFC 1901，RFC 1905和RFC 1906中定义。
  • SNMPv3：鉴于SNMPv2c在安全性方面没有得到改善，IETF颁布了SNMPv3版本，提供了基于USM（User Security Module）的认证加密和基于VACM（View-based Access Control Model）的访问控制，是迄今为止最安全的版本。SNMPv3在RFC 1905，RFC 1906，RFC 2571，RFC 2572，RFC 2574和RFC 2575中定义。

就一般性的网络监控，还是SNMPv2c用的比较多。

SNMP基本结构

SNMP系统包括网络管理系统NMS（Network Management Station）、代理进程Agent、被管对象Management object和管理信息库MIB（Management Information Base）四部分组成。

NMS

NMS是管理端、服务端，简单理解是通过snmp协议对网络设备进行监控、管理的那个东西。

  • NMS可以向设备上的Agent发出请求，查询或修改一个或多个具体的参数值。
  • NMS可以接收设备上的Agent主动发送的Trap信息，以获知被管理设备当前的状态。

Agent

Agent是被监控、管理的那些设备上的一个代理进程，维护着被管理设备上的相关数据信息，然后跟NMS交互 。

  • 当Agent接收到NMS的请求信息时，通过MIB表完成相应指令后，并把操作结果响应给NMS。
  • 当设备发生故障或者其它事件时，设备会通过Agent主动发送信息给NMS，向NMS报告设备当前的状态变化。

MIB

MIB是一个数据库，定义了被管理设备的一系列属性：对象的名称、对象的状态、对象的访问权限和对象的数据类型等，这些就是能够被Agent查询和设置的信息。

SNMP的管理信息库采用和域名系统DNS相似的树型结构，它的根在最上面，根没有名字。如下图所示的是管理信息库的一部分，它又称为对象命名树。每个OID（object identifier，对象标识符）对应于树中的一个管理对象，如system的OID为1.3.6.1.2.1.1，interfaces的OID为1.3.6.1.2.1.2。

通过OID树，可以高效且方便地管理其中所存储的管理信息，同时也方便了对其中的信息进行批量查询。

特别地，当用户在配置Agent时，可以通过MIB视图来限制NMS能够访问的MIB对象。MIB视图实际上是MIB的子集合。

MIB有着标准结构，比如rfc1213, rfc1471 , rfc1724, rfc2618等等，通用性MIB rfc1213习惯称为MIB-II，如果标准信息不够用，我们也可以自定义MIB（大部分情况下用不到，非必要不推荐）。

参考：SNMP-MIB结构与语法，华为S7700 V200R019C10 MIB参考，华三RFC1213-MIB

Managed object

Managed Object指被管理对象。每一个设备可能包含多个被管理对象，被管理对象可以是设备中的某个硬件，也可以是在硬件、软件（如路由选择协议）上配置的参数集合。

SNMP工作模式

当网络中启动SNMP协议，NMS作为整个网络的网管中心，会对设备进行管理。每个被管理设备都包含驻留在设备上的Agent、多个被管对象和MIB，NMS通过与运行在被管理设备上的Agent交互，由Agent通过对设备端的MIB的操作，完成NMS的指令。SNMP的工作原理是将协议数据单元（也称为SNMP GET请求）发送到响应SNMP的网络设备。用户通过网络监控工具可以跟踪所有通信过程，并从SNMP获取数据。

细节部分可参考：华为IP知识百科-什么是SNMP、华为S12700交换机SNMP配置-SNMPv1/SNMPv2c原理、华三S7500X系列交换机snmp配置。

在NMS与Agent交互的过程中，会涉及到一个community团体字的认证，这个团体字可以理解为密码，同时分为只读和可写两部分认证，只读是指管理端仅可以通过snmp获取被管理设备的相关数据；可写是指管理端还可以通过snmp修改被管理端的相关数据。

大部分（也可能是所有）设备默认情况下配置的只读权限团体字是public，可写权限团体字是private，出于安全考虑，建议修改默认团体字。

SNMP端口

SNMP端口是SNMP通信端点，SNMP消息传输通过UDP进行，通常使用UDP端口号161/162。有时也使用传输层安全性（TLS）或数据报传输层安全性（DTLS）协议，端口使用情况如下表所示。

过程 协议 端口号
代理进程接收请求信息 UDP协议 161
NMS与代理进程之间的通信 UDP协议 161
NMS接收通知信息 UDP协议 162
代理进程生成通知信息 - 任何可用的端口
接收请求信息 TLS/DTLS 10161
接收通知信息 TLS/DTLS 10162

Linux配置SNMP

要在linux配置snmp，需要下面几个步骤。

官方文档：redhat7使用 Net-SNMP 监控性能、debian使用snmp。

安装snmp

RHEL系列linux：

1

sudo yum install net-snmp net-snmp-utils

如果只是作为被管理设备，只安装net-snmp就够了，另外一个是提供snmptrap、snmpget或snmpwalk等命令用的。

debian系linux：

1

sudo apt-get install snmp snmpd

snmp是提供查询用的，可选安装。

安装完成之后，可通过如下命令进行启动、停止snmp服务等操作：

配置snmp

默认情况下snmp的community团体字是public。通过命令snmpwalk -v2c -c public localhost就可以看到本机一些基本信息，为了安全考虑，我们需要修改一下这个community，或者再限制一下可查看对象为NMS管理服务器(假设IP为192.168.120.11)。

debian系：

1
2
3
4
5

# sudo vi /etc/snmpd/snmpd.conf
# 大概70行左右，将rocommunity public default -V systemonly修改为如下内容
# 只有nms服务器和本级可以查看。
rocommunity new_pwd 192.168.120.11 -V systemonly
rocommunity new_pwd localhost -V systemonly

rhel系：

1
2
3
4

# sudo vi /etc/snmpd/snmpd.conf
# 大概在40行左右，修改如下
com2sec notConfigUser  192.168.120.11       new_pwd
com2sec notConfigUser  localhost       new_pwd

另外还有几个常用的系统可选配置：syslocation配置设备所在位置；syscontact配置设备管理人员联系方式；sysname配置设备名称，这个一般就是hostname或者ip地址，也在这里可以自定义一下。

其他更多更复杂的配置，比如分群组配置snmp的权限、团体字等，请自行阅读/etc/snmpd/snmpd.conf配置文件的相关说明，或者snmpd.conf的man文档。

windows配置snmp

如果是windows服务器，是在功能里面，添加功能，选择snmp相关功能。如果是windows7、windows10等系统，是在控制面板的程序与功能里，启用或关闭windows功能，选择snmp相关的，但是有的系统好像不带snmp功能。

华为华三交换机配置snmp

就个人目前的接触，交换机配置snmp就是用于zabbix等运维监控平台查看网络设备的系统性能、端口状态等，不需要可写权限去修改交换机的配置参数。

# 配置acl，只允许NMS可以管理交换机（假设nms所在ip为192.168.120.11）
[h3c_sw2]acl basic 2000
[h3c_sw2-acl-ipv4-basic-2000]des snmp_permit_NMS
[h3c_sw2-acl-ipv4-basic-2000]rule per sou 192.168.120.11 0
[h3c_sw2-acl-ipv4-basic-2000]rule deny sou any
[h3c_sw2-acl-ipv4-basic-2000]qu
# 配置snmp版本
[h3c_sw2]snmp-agent sys-info ver all
# 配置团体字为new_com$、访问控制方式、NMS
[h3c_sw2]snmp-agent community simple new_com$ user-role network-operator acl 2000
# 可选，配置设备所在位置
[h3c_sw2]snmp-agent sys-info location ZXJF_A1_41U
# 可选，开启NMS告警功能，告警信息发送到NMS，使用的团体名为public。
[h3c_sw2]snmp-agent trap enable
[h3c_sw2]snmp-agent target-host trap address udp-domain 192.168.120.11 params securityname new_com$ v2
# 配置发送告警的源ip
[h3c_sw2]snmp-agent trap sou vl 10

基本的监控使用，以上配置应该就够了，配置了snmp版本为全版本（或者最长用的v2c），更改团体字名称为new_com$，并通过acl限制仅有nms服务器可访问，nms地址为192.168.120.11，访问控制方式为network-operator，表示所有mib对象可读。

访问控制方式还有network-admin、level-15等，具体含义参考华三官网。

可选配置还设定了交换机所在物理位置，中心机房A1机柜41U。

可选配置还开启了snmp trap，当交换机出现故障的时候，会自动使用trap方式向NMS发送告警，这里的udp-domain后面的ip地址就是指定接收的nms地址，securityname后面的new_com$是nms上配置的接收trap告警的团体字。

同时，由于交换机，特别是核心、汇聚交换机上可能存在多个vlanif地址、loopback地址等，需要指定一个发送trap告警消息的源ip地址，与nms上添加主机监控的ip保持一致。

更多配置参考官方文档：华三S7500X系列交换机snmp配置。

NMS获取常用信息

NMS网管可以使用snmpwalk获取设备的信息，比较常见的如服务器、网络设备的cpu/内存使用率、服务器的磁盘使用情况等信息。这里简单记录一下华三、华为网络设备等获取相关信息的方法。

SNMP Trap

snmptrap是被管理设备主动向NMS管理平台发送的消息。rhel系linux需要安装net-snmp-utils；debian系linux需安装snmp。

1

snmptrap -v2c -c public nms_ip:162 '' UPDATE 1.3.6.1.6.3.1.1.4.1.0 s "Test Trap Message"

public团体字要跟NMS端的相关配置对应。

''是通知类型，留空表示默认类型。

1.3.6.1.6.3.1.1.4.1.0是OID，具体数值根据被管理设备实际情况来。

最后面的字符串表示要发送的消息。
