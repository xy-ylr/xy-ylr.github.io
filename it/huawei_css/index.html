堆叠是指将多台支持堆叠特性的交换机通过堆叠线缆连接在一起，从逻辑上变成一台交换设备，作为一个整体参与数据转发。

集群也类似，将两台支持集群特性的交换机设备组合在一起，从逻辑上虚拟成一台交换设备。

堆叠stack

官方文档：S300, S500, S2700, S5700, S6700 V200R023C00 配置指南-堆叠配置，其他型号交换机的配置也请参考官方文档。

基本概念

堆叠成员

参与堆叠系统的那些交换机，按照功能分为三种角色：

  • 主交换机（master）：负责管理整个堆叠系统，一个堆叠中只有一台master。
  • 备交换机（standby）：备份交换机，当master故障的时候，standby会升级接替master管理堆叠系统，也只有一台。
  • 从交换机（slave）：除master和standby以外的其他交换机，只参与业务数据处理。

另外还有一种单机堆叠，其实就只是一台普通的交换机启用了堆叠功能而已。

堆叠ID

用于标识堆叠成员交换机的唯一ID。可以通过dis stack来查看堆叠ID，回显中的slot部分就是；也可以通过业务口指示灯来判断，具体判断方法参考官网，比如S300, S500, S2700, S5700, S6700 V200R023C00交换机通过指示灯显示堆叠ID。

ID取值范围是0-8，默认为0。

一般在配置堆叠或者堆叠扩容的时候，推荐先手动配置交换机的堆叠ID，然后再用堆叠线将交换机连接起来。

可以使用命令stack slot slot-id renumber [n]来更改堆叠ID。

堆叠优先级

是系统选举过程中确定master主交换机的一个重要参考指标。

优先级取值范围1-255，默认为100。可以使用命令stack slot [slot-id] priority [priority]来修改。

堆叠选举建立过程

将参与堆叠的交换机连接起来，可以是链形连接也可以是环形连接，然后通电启动。第一条件是比较交换机的启动时间，启动最早的成为master。当交换机的启动时间间隔在20秒之内，系统就会当这些交换机是同时启动的，此时堆叠优先级最大的交换机被选举成为master。如果堆叠优先级也一样，那么mac地址小的那个交换机竞选为master。

所以如果想要确保某台交换机成为master，最简单就是优先启动，过个半分钟再启动其他交换机。

选出master之后，如果事先没有手动配置交换机的堆叠ID，此时系统会自动为其他交换机分配堆叠ID。

之后再进一步选举standby，还是按照启动时间、堆叠优先级、mac地址大小来选举。

standby也选出来之后，剩余的其他交换机就都是slave。

最后系统会进一步比较各交换机的软件版本和配置文件，如果有不一样的，都会自动同步master的。

补充：

堆叠建立完成后，如果后续又有新的交换机要加入堆叠，最好是先开机预配置堆叠ID并启用堆叠功能，或者是提前将物理端口加入逻辑堆叠端口，然后关机，将新交换机物理接入堆叠系统后再重新开机。

如果是环形堆叠，其中一台slave交换机出现了故障或者链路断开了，原环形堆叠变成链形堆叠，堆叠系统重新计算下拓扑并同步一下就算完成了；如果是standby退出了，那还需要重新选举standby；如果是master故障退出了，那原来的standby变为master，然后重新选举新的standby，再计算拓扑并同步。

如果是链形堆叠，其中一台交换机故障或链路断开退出，可能会导致堆叠分裂，原master所在的一边重新计算拓扑并同步就好了，另一边的可能会重启并重新进行堆叠系统的选举建立。

如果是分裂出去的堆叠链路修复要重新并入原堆叠系统，或者是有新的堆叠系统要合并到原来的堆叠中，最好的操作是将要合并进来的交换机进行一些预配置后先关机，然后物理线路连接好之后再重新通电，因为两个堆叠系统合并的时候，需要重新竞选master，如果当前堆叠的master竞选失败了，那这一批交换机都会重启，可能会造成大量业务中断。

堆叠IP/MAC

堆叠系统作为一台设备与其他网络设备进行通信时，具有唯一的IP地址和MAC地址。

堆叠的IP地址是全局的，不是针对主交换机或其他堆叠成员交换机。任意成员交换机从堆叠系统中移除，都不会改变堆叠系统的IP地址。

堆叠系统的IP地址可以是任意堆叠成员交换机管理网口或三层端口的IP地址，正常情况下是master的管理口IP，如果master管理口不可用，再选其他的。堆叠管理网口编号与单机一样，都是MEth0/0/1。

堆叠系统的MAC地址正常情况下是主交换机master的MAC地址，但是master断开，则有可能造成堆叠mac切换，默认的时间是10分钟。如果断开超过10分钟，堆叠mac会切换为原来的standby（此时已经升级为新的master了）的mac；如果在10分钟内成功重连了，那么堆叠mac不变（虽然堆叠mac还是重连的交换机的mac，但是成员角色还是变了，原standby升级为master，重连的交换机变为slave或者standby，此时可以使用slave switchover来切换一下交换机成员角色）。

可以使用命令stack timer mac-address switch-delay [delay-time]来修改切换时间。如果时间设置为0，除非堆叠重启，否则mac永远不变。

多主检测

由于堆叠系统中所有成员交换机都使用同一个IP地址和MAC地址（堆叠系统MAC），堆叠分裂的时候，可能会产生多个具有相同IP地址和MAC地址的堆叠系统。这对于整个网络系统来说，就是有IP/MAC地址冲突，可能会造成严重网络故障。

比如分裂的时候master和standby分别在一边，原master所在堆叠重新选举standby并同步拓扑；原standby所在堆叠，standby升级为master并重新选举standby等，没有重新启动交换机选举master，在堆叠mac老化之前，两边堆叠用的都是一样的IP和mac。

多主检测MAD（Multi-Active Detection），是一种检测和处理堆叠分裂的协议。链路故障导致堆叠系统分裂后，MAD可以实现堆叠分裂的检测、冲突处理和故障恢复，降低堆叠分裂对业务的影响。MAD检测有直连检测和代理检测两种方式。

  • 如果参与堆叠的交换机不超过3台，可以使用Full-mesh直连检测，将各交换机两两互连。
  • 因为Full-mesh直连要求每台设备都要互连，如果堆叠交换机超过3台，那需要的端口数实在太多，可以使用中间设备直连检测，将每台堆叠成员交换机与中间设备交换机互连。
  • 如果实在是端口紧张，可以使用代理检测，跟中间设备直连检测类似，只是运行代理检测的端口除了用于mad检测，还可以正常运行业务。代理设备可以是其他堆叠系统，或者其他单台交换机。两者之间的互连线路需要配置Eth-Trunk。受限于每个Eth-Trunk最多只有8个端口，所以堆叠成员最好不要超过8个。

配置mad检测之后，会给堆叠配置Detect状态表示正常工作。如果堆叠分裂了，分裂之后形成的堆叠之间需要相互竞争，成功的继续保持Detect，失败的变为Recovery状态，表示堆叠禁用。Recovery状态的交换机除了保留端口以外的端口都会被down掉。

竞争规则与master选举类似，先比较启动时间（早的优先），再比较主交换机堆叠优先级（高的优先），最后比较堆叠系统MAC（小的优先）。所以大部分情况下应该都是原堆叠系统master所在的这边保持Detect，另外的转为Recovery。

如果导致堆叠分裂的故障修复了，Recovery状态的堆叠交换机会重启并与Detect状态的合并，同时端口也恢复up。

如果故障修复前，Detect状态的堆叠也再次故障了，可以先临时将这部分交换机断线脱网，然后手动启用Recovery状态的堆叠，等所有故障都修复后，再重新合并堆叠系统。

配置流程

使用堆叠卡来组建堆叠

  1. 首先安装堆叠卡。
  2. 再开机配置各成员交换机的堆叠ID、堆叠优先级等。
  3. 保存之后关机，然后连接堆叠线再重新开机。
  4. 检查堆叠配置是否成功。

业务口使用普通网线或光纤线来组建堆叠

  1. 首先配置交换机物理端口加入逻辑堆叠口。
  2. 再配置交换机的堆叠ID、堆叠优先级等。
  3. 保存之后关机，然后连接端口再重新开机。
  4. 检查堆叠配置是否成功。

业务口专用堆叠线组建堆叠

  1. 可以不用预配置，直接连接端口，具体端口参考官方文档。
  2. 然后按照顺序开机。
  3. 检查堆叠是否成功。
  4. save保存专用线缆自动生成的配置。

在连接线缆的时候，都是交换机的1口接另一交换机的2口。

堆叠配置完成后，再配置mad检测，以及堆叠mac切换时间、堆叠保留vlan等。

集群CSS

集群CSS也跟堆叠stack类似，用于将两台支持集群特性的交换机设备组合在一起，从逻辑上虚拟成一台交换设备。只是相对来说集群用于高端交换机，且集群仅指两台虚拟成一台，而堆叠有的甚至可以十几台堆成一台。

基本概念

组建集群的时候，成员交换机也分主（master）备（standby）交换机；也通过集群ID（CSS ID）作为成员交换机的标识；也有集群优先级（CSS Priority）属性用于角色选举。

集群ID默认为1，组建集群的时候需要将其中一台的ID修改为2。集群优先级默认为1，优先级值越大，越可能被选为master。

选举过程也类似，先看启动时间，启动早的成为master；再比较集群优先级，优先级值大的成为master；再比较设备mac地址，mac小的成为master；最后比较集群ID，ID值为1的成为master。

多主检测

由于集群系统中所有成员交换机都使用同一个IP地址和MAC地址（集群系统MAC），一个集群分裂后，由于这两台交换机运行着相同的配置文件（即原集群系统的配置文件），此时就会产生两个具有相同IP地址和MAC地址的单框集群系统。为防止集群分裂后，产生两个具有相同IP地址和MAC地址的集群系统，引起网络故障，必须进行IP地址和MAC地址的冲突检查。

多主检测MAD（Multi-Active Detection），是一种检测和处理集群分裂的协议。链路故障导致集群系统分裂后，MAD可以实现集群分裂的检测、冲突处理和故障恢复，降低集群分裂对业务的影响。检测方式同样分为直连检测方式和代理检测方式，直连检测又可以分为直接直连和中间设备直连。在同一个集群系统中，两种检测方式互斥，不可以同时配置。

如果确认配置无误，接下来就可以连接线缆，然后使用命令css enable使能集群。执行该命令后，交换机会提示css集群配置会在重启后生效，注意先使能并重启SWA交换机，这样才能顺利成为master。

等两台交换机重启完成，集群应该就组建成功了。后续可以使用命令display css channel all查看css集群链路拓扑。
